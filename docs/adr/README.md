# Architecture Decision Records

This directory will hold Architecture Decision Records for decisions that materially affect platform security, deployment, identity, runtime behavior, or project governance.

Use an ADR when a decision:

- changes the security model
- selects a runtime, identity, storage, or deployment pattern
- creates a long-lived public contract
- resolves an open question from an architecture document
- affects how customers operate or trust the platform

Suggested first ADRs:

- [ADR-0001: Runtime strategy](0001-runtime-strategy.md)
- delegated identity and OBO token propagation model
- open-source license selection
- first supported secret provider
- first supported graph backend
- deployment renderer choice
- OpenShell Gateway configuration delivery mode
